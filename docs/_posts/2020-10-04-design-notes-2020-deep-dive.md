Docs aren’t marketing — they’re an operational tool.

## Good docs do 3 things
1. Reduce the number of questions
2. Reduce the time-to-fix
3. Provide a shared language for teams

## Practical takeaway
Write docs in the same repo as code and treat docs changes like code changes.
